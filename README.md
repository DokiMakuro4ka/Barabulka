## 📜 О проекте

Полнофункциональный RPG-бот для Telegram, предоставляющий игрокам возможность:
- Выбрать государство и погрузиться в его историю
- Создать персонажа одного из четырёх классов
- Участвовать в пошаговых боях
- Прокачивать уровень и собирать лут

---

## ✨ Основные возможности

### 🏰 Четыре государства
- **Айкацу** – восточная магия и честь
- **Северное Молчание** – холодные земли и выживание
- **Пылающий Предел** – страна огня и боли
- **Дварфы** – подземные королевства и ремесло

### 📚 Система лора
- Эпохи истории каждого государства
- Детальное описание культуры, конфликтов и магии
- Хранение в базе данных PostgreSQL

### ⚔️ Четыре класса персонажей
| Класс | Характеристика |
|-------|----------------|
| 🧙 **Маг** | Высокий урон магией, низкая защита |
| 🛡️ **Рыцарь** | Высокая защита, средний урон |
| 🏹 **Лучник** | Сбалансированный класс |
| 🗡️ **Ассасин** | Высокий урон, низкая защита |

### 🎮 Боевая система
- Пошаговые сражения с монстрами
- Действия: **Атаковать**, **Защищаться**, **Убежать**
- Система опыта и уровней
- Прибыль: монеты и предметы

### 👤 Профиль игрока
- Просмотр статистики (уровень, HP, MP, атака, защита)
- Управление персонажем
- Удаление профиля с каскадным удалением данных

## 🚀 Установка и запуск

### Требования
- **Python** 3.11 или 3.12 (не 3.14)
- **PostgreSQL** 12+

### Шаг 1: Клонирование репозитория

```bash
git clone https://github.com/DokiMakuro4ka/Barabulka.git
cd Barabulka
```

### Шаг 2: Подготовка базы данных

1. Создайте базу данных в PostgreSQL:
```sql
CREATE DATABASE Barabulka;
```

2. Выполните SQL-скрипт с таблицами и начальными данными (файл будет указан в документации).

### Шаг 3: Создание файла конфигурации

Скопируйте `.env.example` в `.env`:

```bash
cp .env.example .env
```

Или создайте `.env` вручную:

```env
# Бот
BOT_TOKEN=ваш_токен_от_BotFather

# База данных
DB_HOST=localhost
DB_PORT=5432
DB_USER=postgres
DB_PASSWORD=123123
DB_NAME=Barabulka

# Опционально: ID фото для иллюстраций
LORE_PHOTO_ID=
STATE_AIKATSU_PHOTO_ID=
STATE_DWARVES_PHOTO_ID=
STATE_ICE_PHOTO_ID=
STATE_FIRE_PHOTO_ID=
```

### Шаг 4: Установка зависимостей

```bash
# Создание виртуального окружения
python -m venv venv

# Активация (Linux/Mac)
source venv/bin/activate

# Активация (Windows)
venv\Scripts\activate

# Установка зависимостей
pip install -r requirements.txt
```

### Шаг 5: Запуск бота

```bash
python main.py
```

## 🎮 Использование

### Основные команды

| Команда | Действие |
|---------|----------|
| `/start` | Запуск бота, просмотр лора, выбор государства |
| `/menu` | Главное меню |
| `/help` | Справка |

### Главное меню

После создания персонажа доступны кнопки:
- **📊 Профиль** – просмотр статистики
- **⚔️ Сражаться** – начать бой с монстром
- **📜 Выбрать класс** – смена класса (только если персонаж существует)
- **ℹ️ Помощь** – информация о боевой системе

### Процесс создания персонажа

```
1. /start
   ↓
2. Прочитать лор мира
   ↓
3. Нажать "Выбрать государство"
   ↓
4. Выбрать государство (с изучением эпох)
   ↓
5. Подтвердить выбор государства ✅
   ↓
6. Выбрать класс (🧙 / 🛡️ / 🏹 / 🗡️)
   ↓
7. Персонаж создан! → Главное меню
```

### Боевая система

**Начало боя:** `/menu` → **⚔️ Сражаться**

Случайно выбирается монстр из локации государства персонажа.

**Доступные действия:**
- **Атаковать** – нанести урон монстру
- **Защищаться** – входящий урон снижается на 50% в этом ходу
- **Убежать** – 50% вероятность успеха

**Результаты:**

| Исход | Награда |
|------|---------|
| 🏆 **Победа** | Опыт + монеты |
| 💀 **Поражение** | Воскрешение с 50% HP |

**Повышение уровня** (каждые 100 опыта):
- +10 HP
- +2 урон
- +1 защита
- Полное восстановление HP

### Профиль игрока

Команда: `📊 Профиль` в главном меню

Показывает:
- Имя персонажа
- Уровень и опыт
- HP / MP
- Атака / Защита
- Монеты
- Класс и подкласс
- Статистика боёв

**Удаление профиля:**
Удаляет персонажа со всеми связанными данными:
- История боёв
- Инвентарь
- Прогрессия квестов

## 🗄️ База данных

### PostgreSQL 12+

**Основные таблицы:**

```
States                    # Государства мира
├── state_id
├── name
└── location_id

Classes                   # Классы персонажей
├── class_id
├── name
└── description

Subclasses                # Подклассы (специализации)
├── subclass_id
├── class_id
└── name

Epochs                    # Исторические эпохи
├── epoch_id
├── state_id
└── name

EpochDetails              # Лор эпох
├── detail_id
├── epoch_id
└── description

Locations                 # Локации для боёв
├── location_id
├── state_id
└── name

Monsters                  # Враги
├── monster_id
├── location_id
├── name
├── hp
├── attack
└── defense

Users                     # Игроки
├── user_id
├── class_id
├── state_id
├── level
├── exp
├── hp
└── coins

Battles                   # История боёв
├── battle_id
├── user_id
├── monster_id
└── result
```

**Использование:** Синхронный драйвер `psycopg2.binary` для всех запросов.
